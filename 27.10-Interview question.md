06. Hoisting: Its a Js default behaviour of moving all declaration to the top of the current scope.exp-

x = 1;
var x;

More about Hoisting: All variable declaration(var, let, const) are hoisted in js,while var declaration are initialized with undifined but let and const declaration remain uninitialized.

07. Hoisting in function: Only function declaration are hoisted in Js, function expression are not hoisted. Js only hoist declaration, not initialization(assignments).

Below code will work:

displayMessage() ;
function displayMessage() {
    console.log("Hello World");
}

Below code won't work:

displayMessage() ;
var displayMessage = function() {
    console.log("Hello World");
} 

08. Closure?

=> Closure means that an inner function always has access to the variables of its outer function. In the example, we have three function - grantParent(), parent() and child() nested inside each other with variables home, car and scooter respectively. The chile() function will be able to access variables - house and car.

Exp-

function grandParent() {
    let house = "GreenHouse";

    function parent() {
        let car = "Tesla";
        house = "YellowHouse";
        function child() {
            let scooter = "yahamaha";
            console.log("i have", house, car, scooter);
        }
        return child();
    }
    return parent();
}
